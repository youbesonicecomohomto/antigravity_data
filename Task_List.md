# カラオケうた姫 会員証アプリ化 タスクリスト

## 現在の進捗状況
*   **フェーズ:** プロトタイプ（UI画面モックアップ）作成完了
*   **実装済み機能:**
    *   スマートフォン向けブラウザで閲覧可能なHTMLモックアップ（Tailwind CSS）
    *   店舗名（カラオケうた姫）およびピンク系テーマカラーの適用
    *   キャラクター画像の配置（中央上部・カード重ねレイアウト）
    *   動的バーコード機能（EAN-13/JANコード：10秒ごとに自動更新）
    *   ポイント表示、キャンペーンバナー（カルーセルダミー）、メニューボタン群の配置
*   **バックエンド設計案:**
    *   Googleスプレッドシート（約1万件）＋ Google Apps Script (GAS) を利用した簡易API構成で合意済み

---

## 次に行うべき未完了タスク（バックエンド連携のテスト）

UI（見た目）は概ね完成したため、次は**「このスマホアプリが、実際に会員データと通信できる仕組み作り（バックエンド）」**に進みます。

### Phase 2: バックエンドAPIの検証と構築
- [ ] **Googleスプレッドシートの準備**
    - テスト用の会員情報を入れたスプレッドシートを作成する。
    - （会員番号、氏名、現在のポイント、電話番号等）を定義する。
- [ ] **Google Apps Script (GAS) APIの開発**
    - スマホから「会員番号〇〇のポイントを教えて」とアクセスされたときに、スプレッドシートの情報を返すプログラム（API）を作成する。
    - ポイントの増減（加算・使用）を受信するプログラムを作成する。
- [ ] **連携テスト用の1万件データ負荷テスト**
    - ダミーで1万行のデータを用意し、検索スピード（店舗のレジ前でバーコードを読み取った際、何秒で情報が返ってくるか）を計測・検証する。

### Phase 3: アプリへの組み込み（フロントエンド更新）
- [ ] **現在のモックアップに通信処理（JavaScript）を追加する**
    - 画面を開いたときに、裏側でGASのAPIへアクセスして実際のポイント数を表示するように修正する。
    - スマホ側で計算した「動的JANコード」の裏側の情報を、サーバーにも送信する（なりすまし照合用）。

### Phase 4: 本格的なスマートフォンアプリ化（ネイティブ化）
- [ ] **フレームワーク（Flutter等）への移植検討**
    - 現在のHTMLモックアップを元に、iOS / Android 両対応のアプリ（Flutter または React Native）に書き直す。
    - これにより、スマホのカメラを使った「旧カードのバーコード読み取り」や「おサイフケータイ機能」「プッシュ通知」が利用可能になる。
